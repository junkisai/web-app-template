このスキルは、ありきたりな「AI っぽい」見た目を避けた、独創的でプロダクションレベルのフロントエンドインターフェースの作成を導く。美的細部とクリエイティブな選択に徹底的にこだわった、実際に動作するコードを実装する。

ユーザーはフロントエンドの要件を提供する：構築するコンポーネント、ページ、アプリケーション、またはインターフェース。目的、対象ユーザー、技術的制約に関するコンテキストを含む場合もある。

## デザイン哲学

優れたデザインは「美を創ろうとすること」よりも、**「醜さへの不寛容」**から生まれる。何かを見て「もっとうまくできる」と感じる感覚——それが優れた仕事の出発点だ。

- **シンプルは本物の証** — 装飾は思考の代替にならない。シンプルを強いられるとき、本当の問題と向き合わざるを得ない。複雑さが美しさを演じているなら、それは逃避だ。
- **時代を超える** — トレンドは定義上、変化する。「10年後もこのデザインは良く見えるか？」を問う。流行ではなく本質に従うとき、作品は merit で評価される。
- **正しい問題を解く** — 美しいことより、正しい問題を解くことが先だ。ユーザーが実際に直面している問題を解かずに視覚的に洗練されたものを作っても意味がない。
- **示唆する、決定しない** — 優れた設計は、ユーザーが自分なりの使い方を発見できる余白を残す。使い方を過剰に定義されたインターフェースは、建築家のプログラムを実行させられているようなものだ。
- **再設計は前提** — 初稿に執着しない。不満を育てる。正確な一本の線のために 5〜6 回の試行を惜しまない。最終的な形は再設計から生まれる。
- **奇妙さを潰さない** — 最高の仕事には不思議な質がある。奇妙さは意図的に生み出せないが、現れたときに潰してはいけない。正しいものを目指すと、真実は奇妙に見えることがある。
- **ユーモアは強さの証** — 自信のあるデザインは自分を笑える。重々しく自分を主張するより、余裕を持って笑いを含む方が強い。ユーモアは威圧しない信頼感だ。
- **難しさが洗練を生む** — 厳しい制約は装飾を削ぎ落とす。予算が少ない、サイトが難しい——そういう条件がエレガントなデザインを強いる。制約のない状況では、余分なものが紛れ込む。
- **簡単に見えること** — 熟練は自然に見える。優れた UI は使うのが当然に感じられる——その背後に何百時間の試行錯誤があっても。この「見かけの自然さ」は省略ではなく練習の産物だ。
- **対称性と再帰** — 対称性はシンプルさを達成する一つの手段だ。反復、再帰、構造の繰り返し——自然もこれを多用する。ただし、対称性を思考の代替にしてはいけない。
- **自然に学ぶ** — 自然は問題を長い時間をかけて解いてきた。自然に似た答えは良い兆候だ。コピーは恥ではない——自然から学ぶことは、最も長い歴史を持つデザインリサーチだ。
- **コピーを恐れない** — 初心者は知らずに模倣し、次に意識的な独創性を求め、最終的に「正しいことが独創性より先」に気づく。誰かがすでに正しい答えを出しているなら、それを使うことを恐れない。
- **大胆に異端であれ** — 常識と真実がずれている場所に注目する。通説に従った安全な選択は新しい領域を開拓しない。今日の実験的な選択が明日の標準になる。

## デザイン思考

コーディングの前に、コンテキストを理解し、**大胆な**美的方向性にコミットする：
- **目的**: このインターフェースはどんな問題を解決するか？誰が使うのか？
- **トーン**: 極端な方向性を選ぶ：徹底的なミニマル、マキシマリストのカオス、レトロフューチャー、オーガニック/ナチュラル、ラグジュアリー/洗練、プレイフル/トイ風、エディトリアル/マガジン風、ブルータリスト/ロウ、アールデコ/ジオメトリック、ソフト/パステル、インダストリアル/ユーティリタリアンなど。無限の方向性がある。これらをインスピレーションとして使いながら、美的方向性に忠実なデザインを作る。
- **制約**: 技術的要件（フレームワーク、パフォーマンス、アクセシビリティ）。
- **差別化**: 何がこれを**忘れられない**ものにするか？人が最も印象に残る一つのことは何か？

**重要**: 明確なコンセプトの方向性を選び、精密に実行する。大胆なマキシマリズムも洗練されたミニマリズムも機能する——鍵は強度ではなく意図性だ。

その後、以下の条件を満たす動作するコード（HTML/CSS/JS、React、Vue など）を実装する：
- プロダクションレベルで機能的
- 視覚的に印象的で記憶に残る
- 明確な美的視点を持ち一貫性がある
- すべての細部が丁寧に磨かれている

## フロントエンド美学ガイドライン

以下に注力する：
- **タイポグラフィ**: 美しく、ユニークで、興味深いフォントを選ぶ。Arial や Inter などの汎用フォントを避け、フロントエンドの美学を高める個性的な選択をする。予想外でキャラクターのあるフォントの組み合わせ。印象的なディスプレイフォントと洗練されたボディフォントをペアリングする。
- **カラーとテーマ**: 一貫した美学にコミットする。一貫性のために CSS 変数を使用する。シャープなアクセントを持つドミナントカラーは、おとなしく均等に分散されたパレットより優れている。
- **モーション**: エフェクトとマイクロインタラクションにアニメーションを使用する。HTML では CSS のみのソリューションを優先する。React では利用可能な場合 Motion ライブラリを使用する。高インパクトな瞬間に集中する：段階的なリビール（animation-delay）による一つのよく組み立てられたページロードは、散在するマイクロインタラクションより多くの喜びをもたらす。スクロールトリガーとサプライズを与えるホバーステートを使用する。
- **空間構成**: 予想外のレイアウト。非対称性。重なり。対角線的なフロー。グリッドを破る要素。寛大なネガティブスペース、または制御された密度。
- **背景とビジュアルディテール**: ソリッドカラーへのデフォルトではなく、雰囲気と奥行きを作る。全体的な美学に合ったコンテキスト固有のエフェクトとテクスチャを追加する。グラデーションメッシュ、ノイズテクスチャ、ジオメトリックパターン、レイヤード透明度、ドラマチックシャドウ、デコレーティブボーダー、カスタムカーソル、グレインオーバーレイなどのクリエイティブな表現を適用する。

過剰に使われているフォントファミリー（Inter、Roboto、Arial、システムフォント）、陳腐なカラースキーム（特に白背景上のパープルグラデーション）、予測可能なレイアウトとコンポーネントパターン、コンテキスト固有のキャラクターを欠いた量産型デザインなど、ありきたりな AI 生成の美学は**絶対に使わない**。

クリエイティブに解釈し、そのコンテキストのために真に設計されたと感じられる予想外の選択をする。どのデザインも同じであってはならない。ライトとダークテーマ、異なるフォント、異なる美学を使い分ける。世代をまたいで共通の選択肢（例えば Space Grotesk）に収束することは**絶対にしない**。

**重要**: 実装の複雑さを美的ビジョンに合わせる。マキシマリストデザインには、広範なアニメーションとエフェクトを伴う精巧なコードが必要だ。ミニマリストまたは洗練されたデザインには、抑制、精度、スペーシング・タイポグラフィ・微妙な細部への丁寧な注意が必要だ。エレガンスはビジョンをうまく実行することから生まれる。

覚えておこう：Claude は並外れたクリエイティブな仕事ができる。固定観念を超えて考え、独自のビジョンに完全にコミットするときに、何が本当に生み出せるかを見せよう。
