# Idea Scout — 穴場ビジネスアイデア発掘スキル

## 目的

個人開発者が「作る価値のあるもの」を見つけるのを助ける。
大企業ではなく個人・小チームが勝てる隙間市場（niche）を、実際のペインポイントから発掘する。

---

## リサーチ手順

### Step 1: トレンド収集（Web 検索）

以下の観点で検索し、生の情報を収集する。検索は複数回行い、幅広くカバーすること。

**検索クエリの例（状況に応じてアレンジして）:**
- `Reddit "is there an app" OR "wish there was" site:reddit.com 2024 2025`
- `Indie Hackers profitable solo founder SaaS niche 2025`
- `Product Hunt trending tools 2025 indie developer`
- `"no good solution" OR "frustrated with" developer tools reddit`
- `underserved market B2C SaaS 2025`
- `micro SaaS niche opportunity`

**収集する情報の種類:**
- Reddit スレッド：「こういうツールがあれば…」「既存サービスの○○が不満」
- Indie Hackers：小規模で黒字化した SaaS の事例（月次収益・ユーザー数など）
- Product Hunt：最近ローンチされた個人開発プロダクト（特に"Top of the Day"クラス）
- GitHub Trending：注目されているOSS（商用化の余地を探る）

### Step 2: アイデア選定フィルタ

以下の基準で厳選する。全部を満たす必要はないが、多く当てはまるほど良い：

| 基準 | 判定ポイント |
|------|------------|
| 実際のペインが存在する | Reddit 等でリアルな不満・要望が確認できる |
| 大企業が参入していない | 検索結果に大手（Google, Microsoft, Adobe 等）の専用ツールがない |
| 個人でも実装できる規模 | MVP を数ヶ月で作れる機能セット |
| B2C または SaaS として成立 | 定期収益または課金の可能性がある |
| Web/モバイルアプリとして実現可能 | デスクトップ専用や専用ハードが不要 |

### Step 3: 各アイデアの深掘り

選定した5件それぞれについて、追加検索を行って情報を補完する：
- 競合サービス名・URL・月額料金・評価
- 市場規模（TAM/SAM/SOM の概算でよい）
- ユーザーがすでに使っているワークアラウンド（代替手段）

---

## 出力構成

実行後、以下のファイルを生成する：

```
idea-scout-<テーマやトピック名>/
├── REPORTS.md          ← 5件のサマリー
├── idea-1-<name>.md    ← 各アイデアの詳細
├── idea-2-<name>.md
├── idea-3-<name>.md
├── idea-4-<name>.md
├── idea-5-<name>.md
└── competitors.csv     ← 競合一覧
```

出力先ディレクトリは、ユーザーが指定した場合はそこに、指定がなければ **現在の Git リポジトリのルート**に、ビジネスアイデアのテーマや内容がひと目でわかるような命名（例: `idea-scout-sns-tools/`, `idea-scout-dev-tools/` など）でフォルダを作成する。
リポジトリルートは `git rev-parse --show-toplevel` で取得する。Git リポジトリ外で実行された場合はカレントディレクトリにフォールバックする。

## 各アイデア詳細ファイル（idea-N-name.md）テンプレート

REPORTS.md の内容をさらに深掘りしたもの。以下を追加で記載する：
- ユーザーインタビュー候補（どこで聞けるか）
- 参考にすべき Indie Hackers 事例（URL付き）
- ランディングページのコピー案（ヘッドライン1文）
- 最初の100ユーザーをどこで獲得するか
- 開発ロードマップ（Phase 1〜3）

```csv
1,アイデア名,競合名,https://...,29,..,...,...
```

---

## 注意事項

- 情報は必ず Web 検索で確認する。知識だけで推測しない。
- 競合の価格・機能は実際のサイトを確認してから記載する。
- 「なぜ個人開発者が勝てるか」の根拠を必ず1つ以上書く。
- 市場規模は大まかな概算でよいが、根拠（検索ボリューム・業界レポートなど）を示す。
- アイデアは現在のトレンドから選ぶ。古い情報（2022年以前）に頼りすぎない。
