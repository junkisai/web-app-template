# commit スキル

変更内容を分析して日本語・絵文字プレフィックス付きのコミットメッセージを自動生成し、コミットを実行する。

---

## 手順

### Step 1: 状態確認

```bash
git status
git diff --staged   # ステージ済みがあれば
git diff            # 未ステージの変更があれば
```

### Step 2: ステージング

- **すでにステージされているファイルがある場合** → そのスコープで進める（`git add` は不要）
- **ステージされていない場合** → 変更内容・ファイルパス・コミットの目的を踏まえて関連ファイルを個別に指定してステージする

  ```bash
  git add <file1> <file2> ...   # 具体的なファイル名で指定（git add . は使わない）
  ```

  機密ファイル（`.env`、クレデンシャル等）は絶対にステージしない。

### Step 3: 差分を読む

```bash
git diff --staged
```

変更の目的・範囲をここで把握する。

### Step 4: コミットメッセージを生成する

差分の内容から **絵文字プレフィックス + 日本語** でメッセージを作る。

#### 絵文字の選び方

| 絵文字 | 用途 |
|--------|------|
| ✨ | 新機能・機能追加 |
| 🐛 | バグ修正 |
| ♻️ | リファクタリング |
| 📝 | ドキュメント・コメント |
| 🔧 | 設定ファイル・ツール設定 |
| 🎨 | コードスタイル・フォーマット |
| ⚡ | パフォーマンス改善 |
| 🔒 | セキュリティ修正 |
| 🗑️ | ファイル・コード削除 |
| 🧪 | テスト追加・修正 |
| 📦 | パッケージ・依存関係 |
| 🚀 | デプロイ・リリース関連 |

#### メッセージの構成

```
<絵文字> <変更の要約（1行・命令形）>

<必要なら詳細を箇条書き>
```

**例：**
```
✨ ユーザー認証機能を追加

- JWT によるセッション管理を実装
- ログイン・ログアウトエンドポイントを追加
```

```
🐛 ダークモード時のボタン色が見えない問題を修正
```

```
🔧 Ghostty のフォントサイズ設定を更新
```

### Step 5: コミット実行

HEREDOC を使ってコミットする。フックは絶対にスキップしない。

```bash
git commit -m "$(cat <<'EOF'
✨ コミットメッセージをここに

詳細があればここに記載

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
EOF
)"
```

### Step 6: 結果確認

```bash
git log --oneline -3
```

コミットが作成されたことを確認してユーザーに報告する。

---

## 注意事項

- **push はしない**。ユーザーから明示的に求められない限り、コミットのみ実行する。
- **既存コミットを amend しない**。常に新しいコミットを作成する（ユーザーから明示的に amend を求められた場合のみ例外）。
- **`git add -A` や `git add .` は使わない**。ファイルを個別に指定してステージする。
- **フック（`--no-verify`）はスキップしない**。フックが失敗した場合は原因を調査・修正してから再コミットする。
- **機密情報を含むファイルはコミットしない**（`.env`、パスワード、APIキー等）。疑わしい場合はユーザーに確認する。
