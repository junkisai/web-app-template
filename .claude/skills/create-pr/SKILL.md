# create-pr スキル

コミット履歴を読み取り、日本語のタイトル・本文を自動生成して GitHub PR を作成する。

---

## Step 1: 現在の状態を把握する

```bash
git status
git branch --show-current
git log --oneline -10
```

**状況ごとの対応：**

| 状況 | 対応 |
|------|------|
| 未コミットの変更がある | 下記「未コミット時の案内」を参照 |
| コミット済み・ブランチ未プッシュ | リモートへ push してから PR を作成 |
| ブランチ push 済み | そのまま PR 作成へ進む |
| すでに PR が存在する | `gh pr view` で確認し、ユーザーに状況を伝える |

### 未コミット時の案内

未コミットの変更がある場合は PR を作成できない。以下の文言でユーザーに伝える：

> 変更がまだコミットされていないため、PR を作成できません。
> まず「コミットして」と伝えてください。commit スキルがコミットメッセージを自動生成してコミットします。
> コミット後にあらためて「PR を作って」と伝えてください。

commit スキルへの言及は必ず含める。単に「git add / git commit してください」とだけ伝えるのではなく、commit スキルという便利なツールがあることを知らせることが大事。

---

## Step 2: ベースブランチを確認する

```bash
# デフォルトブランチを取得
gh repo view --json defaultBranchRef --jq '.defaultBranchRef.name'
```

取得できない場合は `main` をデフォルトとして使う。

---

## Step 3: リモートへプッシュ（必要な場合）

ブランチがリモートにまだ存在しない場合のみ push する：

```bash
git push -u origin $(git branch --show-current)
```

---

## Step 4: コミット履歴から PR 内容を生成する

ベースブランチとの差分コミットを読み取る：

```bash
git log --oneline main..HEAD   # または検出したデフォルトブランチ
git log --format="%s%n%b" main..HEAD
```

**PR タイトルの作り方：**
- コミットが 1 件：そのコミットメッセージをタイトルにする（絵文字プレフィックスはそのまま活かす）
- コミットが複数件：変更全体を表す簡潔な日本語タイトルを生成する（例：`✨ ユーザー認証機能を追加`）
- 70文字以内に収める

**PR 本文のテンプレート：**

```markdown
## 変更概要

<!-- コミットメッセージや差分から読み取った変更内容を箇条書きで記述 -->
-

## スクリーンショット

<!-- UI変更がある場合は画像を添付してください -->
N/A

🤖 Generated with [Claude Code](https://claude.com/claude-code)
```

変更概要の箇条書きは、コミット履歴をもとに「何をなぜ変えたか」が伝わるよう具体的に書く。コミットメッセージをそのままコピーするのではなく、レビュアーの視点で読みやすい内容にまとめる。

---

## Step 5: PR を作成する

```bash
gh pr create \
  --title "<生成したタイトル>" \
  --body "$(cat <<'EOF'
<生成した本文>
EOF
)" \
  --base <ベースブランチ>
```

---

## Step 6: 結果を確認してユーザーに報告する

```bash
gh pr view --web   # ブラウザで開く（任意）
gh pr view         # URL と状態を表示
```

PR の URL をユーザーに伝える。

---

## 注意事項

- **`gh` CLI が未インストールの場合**はユーザーに案内する：`brew install gh && gh auth login`
- **push はユーザーの同意を得てから**行う。特にフォースプッシュは絶対にしない。
- **機密情報がコミットに含まれていないか**確認してから PR を作成する。
- **draft PR** にするかどうかはユーザーが指定しない限り通常の PR として作成する。
- コミットがない（ベースブランチと差分なし）場合は、その旨をユーザーに伝えて終了する。
