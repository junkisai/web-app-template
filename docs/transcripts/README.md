# transcripts

ボイスレコーダーの文字起こしデータ置き場。打ち合わせ・発想メモ・ヒアリングなど種類を問わず、録音から書き起こしたテキストをここに集約する。「いつ・誰が・何を話したか」を後から検索・振り返りできる状態を目指す。

## 命名規則

- 1録音 = 1ファイル。ファイル名は `YYYY-MM-DD-<topic>.md`。
  - `YYYY-MM-DD` は録音日。`<topic>` は内容がわかる短い kebab-case（英数字とハイフン）。
  - 例: `2026-06-29-elink-kickoff.md`
- 同じ日に複数ある場合は末尾に連番を付ける（`2026-06-29-elink-kickoff-2.md`）。

## 書き方

- 新規作成は [`_template.md`](_template.md) をコピーして使う。
- 先頭の frontmatter で録音のメタ情報を記録する。種別（meeting / memo / interview / other）はここで区別する。
- 本文は **要約 → ネクストアクション（任意）→ 生文字起こし** の順。
  - **要約**: 後で全文を読まなくても要点がつかめるように箇条書きで。
  - **ネクストアクション**: 決まった TODO があれば。なければ節ごと削ってよい。
  - **生文字起こし**: 文字起こしツールの出力をそのまま貼る。整形は最小限でよい。

## 音声ファイルの扱い

- 音声ファイル（m4a / mp3 / wav など）は **Git にコミットしない**（[`.gitignore`](.gitignore) で除外）。
- 音声本体は外部（Google Drive 等）に置き、frontmatter の `audio` にリンクまたはファイル名を記録する。
